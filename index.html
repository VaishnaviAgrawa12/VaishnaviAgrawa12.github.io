Purchase Decision API

Live

REST API that scores a purchase against income, savings goals, and spending history, returning a verdict, score, and savings timeline. Design decision: a deterministic rule engine produces the verdict; a separate LLM layer writes the explanation, with a rule-based fallback so the endpoint always returns a valid result even when the model call fails. Deployed on Railway with CI/CD; verified the fallback by breaking the LLM call on purpose.
